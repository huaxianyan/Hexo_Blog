从 Riru 迁移到 Zygisk

前不久 Magisk 发布了最新的 Release，标志着 Magisk 正式进入 Zygisk 时代。

而 Riru 也发布了最后一个 Release，宣告着它的结束。

我打开自己的手机整理了下，发现所有 Riru 的模块都有了 Zygisk 版本。

那么是时候告别 Riru，迎接 Zygisk 了。

首先是核心模块 Riru，直接禁用，然后去 Magisk App 设置中打开 Zygisk。

然后就是 存储空间隔离，以前也叫「存储重定向」，是为了应对国内软件随地大小便的情况而使用的。将 Riru 版的增强模块 Riru - Enhanced mode for Storage Isolation 换成 Zygisk 版的 Zygisk - Enhanced mode for Storage Isolation。

再之后是 Sui，禁用 Sui 换成 Zgyisk - Sui。

帮助手机通过 SafetyNet 验证的 Universal SafetyNet Fix，也从 Riru 版 换成 Zygisk 版。

还有可以微信支持指纹支付的 FingerprintPay，在 Release 中选择 Zygisk 版安装。

因为我要使用 Google 地图的时间线功能，所以 Location Report Enabler 还是需要的，Riru - Location Report Enabler 因为 Google 现在位置记录不再锁中国 Sim 卡，所以项目已经扬了，但是还有个 Magisk 版的 Location Report Enabler，虽然不如 Riru 只针对几个需要的进行改动，但是可以用。

2024.06.17 更新：找到新的解决方法，详情看 /3pYN3iiwkHmbWQ.html

2022.03.04 更新：这个模块虽然可以启用时间线功能，但是会干扰到开启 Google 通话自带录音功能的 mod「GoogleDialerMod」，并且也会导致 Google 应用的天气功能失效，所以综合考虑最后放弃了时间线功能。

最后 Magisk 进入 24.0 以后除了带来了 Zygisk，也放弃了 Magisk Hide，虽有排除列表，但是和 Hide 不是一回事。

为了应对国内多管闲事的 App，还需要安装 Shamiko。使用方法很简单，用 Magisk App 中 Zygisk 排除列表选择要针对隐藏的 App，选择完毕后再关闭排除列表开关，重新启动即可生效。

一套换下来之后，使用起来几乎没有区别，这无疑是最好的结果。

和 Riru 挥挥手，再见了，老朋友。
